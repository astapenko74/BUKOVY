## 2. Игровое поле

### 2.1. Ввод буквы — `cell-pop`

- **Триггер:** класс `.cell.filled` на ячейке с буквой.
- **Длительность:** 100 ms, `ease`.
- **Ключевые кадры:** scale 1 → 1.04 (50%) → 1.

### 2.2. Переворот ячейки при проверке слова — `revealRow()`

**Стадии (на каждую ячейку в строке):**

1. Задержка старта: `col × 150 ms` (слева направо).
2. Добавление класса `.flipped` на `.cell-inner`.
3. CSS-transition: **750 ms**, `cubic-bezier(0.4, 0, 0.15, 1)`, `rotateY(180deg)`.
4. После завершения всех ячеек строки — обновление клавиатуры + **120 ms** пауза до разблокировки ввода.

### 2.3. Ошибка дубликата слова — `shake`

- **Триггер:** класс `.grid-row.shake`.
- **Длительность:** 600 ms, `ease`.
- **Движение:** `translateX` −4px / +4px (чередование).
- Сообщение об ошибке снимается через **600 ms**.

### 2.4. Победа в строке — `cell-win-scale`

- **Триггер:** класс `.cell.win-scale` на ячейках выигрышной строки.
- **Длительность:** 1000 ms, `ease`.
- **Задержка:** `calc(var(--win-col) × 100 ms)` — волна слева направо.
- **Ключевые кадры:** 1 → 1.15 (30%) → 1.05 (40%) → 1.1 (50%) → 1.
- **Общая длительность JS-ожидания:** `(COLS − 1) × 100 + 1000` = **1400 ms**.

### 2.5. Tooltip дубликата

- Появление/скрытие: **200 ms**, `opacity` + `visibility` + `transform`.
- Автоскрытие: **1800 ms** (по умолчанию).

## 3. Клавиатура

### 3.1. Смена состояния клавиши

- `background`, `border-color`: **380 ms** `ease-out`.
- `color`: **200 ms** `ease-out`.

### 3.2. Нажатие

- `transform: scale(0.96)` — **80 ms** (без отдельного transition, мгновенный active).

## 4. Экран результата

### 4.1. Общая последовательность при победе

```
revealRow → playWinScaleAnimation (1400 ms) → playNormalWordAnimation (1000 ms)
→ playWinHeroAnimation (800 ms, параллельно с прогрессом)
```

При поражении (6 попыток): без `playWinScaleAnimation` и без конфетти.

---

### 4.2. Переход к результату — `playNormalWordAnimation()`

**Длительность:** `NORMAL_WORD_MS` = **1000 ms**.  
**Класс:** `.is-animating-normal-word` → затем `.scenario-normal-word`.

| Элемент | Свойство | Длительность | Easing |
|---------|----------|--------------|--------|
| `.grid-area` | `margin-top` | 1000 ms | `cubic-bezier(0.45, 1.45, 0.8, 1)` |
| `.grid`, `.grid-row` | `gap` | 1000 ms | то же |
| `.cell` | `width`, `height` | 1000 ms | то же |
| `.cell-front`, `.cell-back` | `font-size`, `border-radius`, `color`, `background` | 1000 ms | то же |
| `.win-panel` | `transform` (slide up) | 1000 ms | то же |
| `.keyboard` | `transform`, `opacity` | 1000 ms / 600 ms | то же |
| `.win-blocks` | `opacity` | 1000 ms | то же |

**Параллельно:**

- Клавиатура уезжает вниз (`--kb-hide-y`).
- Панель победы поднимается с `--win-panel-offset`.
- Сетка сжимается до миниполя: ячейки **23×22 px**, gap **2 px**, буквы `color: transparent`.
- Конфетти стартует на **20%** анимации (**200 ms**), только при победе.

---

### 4.3. Hero-блок — `playWinHeroAnimation()`

**Длительность:** `WIN_HERO_MS` = **800 ms**.  
**Классы:** `.is-animating-win-hero` → `.scenario-win-hero`.

| Элемент | Свойство | Длительность | Easing |
|---------|----------|--------------|--------|
| `.win-word`, `.win-message`, `.win-share` | `margin-top`, `opacity` | 800 ms | `cubic-bezier(0.45, 1.45, 0.8, 1)` |
| `.ad-badge` | `opacity` | 800 ms | то же |

**Стадия 1** (только `.scenario-normal-word`): hero невидим, место зарезервировано.  
**Стадия 2** (`.scenario-win-hero`): hero появляется в финальной позиции.

Параллельно запускается `animateWinProgress()` или `animateFifthWordProgress()`.

---

### 4.4. Конфетти — `playConfetti()`

- **Частиц:** 200.
- **Цвета:** `#ffdd2d`, `#19b1ff`, `#7d3bc0`, `#ffb400`.
- **Спавн:** слева и справа, по вертикали между низом сетки и hero-слотом.
- **Физика (на кадр):** скорость 4.5–9.5, гравитация 0.11–0.18, decay 0.005–0.009.
- **Рендер:** `requestAnimationFrame`, без фиксированной длительности — до исчезновения частиц.

---

### 4.5. «Играть ещё» — обратные анимации

#### 4.5.1. `playReverseWinHeroAnimation()` — 800 ms

1. Скролл `main` наверх за **800 ms** (easing: `1 − (1−t)³`).
2. Скрытие hero: снятие `.scenario-win-hero`, класс `.is-animating-reverse-win-hero`.
3. **Unflip миниполя** — `unflipAllCells()` (см. §4.6).

#### 4.5.2. `playReverseNormalWordAnimation()` — 1000 ms

**Классы:** `.is-animating-reverse-normal-word` → `.is-returning-to-game`.

| Элемент | Easing |
|---------|--------|
| Сетка, ячейки, панель, клавиатура, блоки | `cubic-bezier(0.7, 0, 0.3, 1)`, **1000 ms** |

На стадии `.is-returning-to-game` сетка возвращается к полному размеру (`--cell-width/height`), клавиатура и панель — в исходные позиции.

---

### 4.6. Unflip миниполя — `unflipAllCells()`

**Когда:** нажатие «Играть ещё», внутри `playReverseWinHeroAnimation`.  
**Общая длительность:** **800 ms**.

**Порядок:** диагональные волны (как при заливке по диагонали):

| Волна | Ячейки (row, col) |
|-------|-------------------|
| 0 | (0,0) |
| 1 | (0,1), (1,0) |
| 2 | (0,2), (1,1), (2,0) |
| … | … |
| 9 | (5,4) |

**Параметры:**

- Старт волны `d`: `round(d × 48.89 ms)`.
- Переворот одной ячейки: **360 ms**, `cubic-bezier(0.4, 0, 0.15, 1)`.
- CSS: `.scenario-normal-word.is-animating-reverse-win-hero .cell-inner`, переменная `--mini-unflip-ms`.
- Буквы очищаются **до** снятия `.flipped`.
- Анимируются только ячейки с классом `.flipped`.

## 5. Прогресс победы (сценарии «2–4 слово» и «5 слово»)

### 5.1. Общая длительность фаз по слоту

| Слот | Phase 1 | Phase 2 | Phase 3 | **Total** |
|------|---------|---------|---------|-----------|
| 0 | — | 420 ms | 380 ms | **800 ms** |
| 1–3 | 280 ms | 420 ms | 380 ms | **1080 ms** |

### 5.2. Фазы `animateWinProgress(word)`

1. **Phase 1** (слоты 1–3): линия до бейджа → 100%, **280 ms**.
2. **Phase 2:** заливка бейджа (scaleX 0→1), **420 ms**. Опционально: стикер тематического слова.
3. **Phase 3:** линия после бейджа → 50%, **380 ms**.

**CSS transitions при `.is-transitioning`:**

| Элемент | Быстрый | Медленный (`.is-transitioning--slow`) |
|---------|---------|---------------------------------------|
| `.win-progress__line-fill` | 280 ms | 380 ms |
| `.win-progress__badge-fill` | 280 ms | — |
| `.win-progress__badge-text` (color) | 420 ms | — |
| `.win-progress__prize-anchor-fill` | 280 ms / 420 ms | — |

Easing фаз прогресса: `cubic-bezier(0.45, 1.45, 0.8, 1)`.

### 5.3. Анимация слова под бейджем

Расчёт от `getProgressTotalMs(slot)`:

- **Точки** (`.win-progress__placeholder--collapse`): старт на **5%** total, длительность **25%** total.
- **Слово** (`.win-progress__word--enter`): старт после точек, видимость до **95%** total.
- Easing слова: `cubic-bezier(0.45, 1.45, 0.8, 1)`.
- Начальное состояние слова: `opacity: 0`, `scale(0.8)`.

**Пример для слота 0 (800 ms):** точки с 40 ms, слово с ~240 ms.

### 5.4. Тематический стикер — `thematic-sticker-attach`

- **Длительность:** 420 ms.
- **Easing:** `cubic-bezier(0.45, 1.45, 0.8, 1)`.
- **От:** `opacity: 0`, `scale(0.35)`, `rotate(-14°)` → **к:** `opacity: 1`, `scale(1)`, `rotate(0)`.

### 5.5. Приз на прогресс-баре

#### Появление (reveal)

- Классы: `.is-revealing` → `.is-revealed`.
- **700 ms**, `cubic-bezier(0.33, 1, 0.68, 1)`.
- `opacity: 0 → 1`, `scale: 0.5 → 1`.

#### Pop (пятисловный сценарий)

- Класс `.is-popped`: **scale 1.2**, **420 ms**, `cubic-bezier(0.33, 1, 0.68, 1)`.

#### Idle-покачивание — `wigglePrize()`

- Интервал: **7000 ms**.
- Анимация: `prize-wiggle-overlay` / `prize-wiggle-panel-layer`, **500 ms**, `ease-in-out`.
- Параллельно: частицы `spawnPrizeParticles()`.

#### Частицы приза — `prize-particle`

- **Длительность:** 0.9 s (JS: 0.7–1.15 s случайно).
- **Easing:** `ease-out`.
- Разлёт по CSS-переменным `--tx`, `--ty`.
- Цвета: `#ffdd2d`, `#ffb400`, `#ffcd33`, `#ffe566`.

#### Полёт к табу «Подарки» — `flyPrizeToGiftIcon()`

- **Длительность:** **1200 ms**.
- Траектория: квадратичная Bézier, контрольная точка выше середины.
- **Scale:** 1.2 → 0.5.
- **Opacity:** 1 до 40% пути, затем fade до 0.
- Easing позиции: `1 − (1−t)³`.
- Иконка таба: `gift-prize-landing`, **480 ms**, scale 1 → 1.2 (42%) → 1, срабатывает на **60%** полёта.

#### Settle нижнего слоя приза

- `.is-settling`: **420 ms**, scale 1.2 → 1.

### 5.6. Сценарий «5 слово» — `animateFifthWordProgress()`

Последовательность после победы:

1. Анимация слова «ВКЛАД» в колонке приза (как слот 3).
2. Линия 3 → 100% (**420 ms** slow transition).
3. Коннектор → 100%.
4. Anchor fill → 1.
5. **Pop** приза + частицы (scale **1.7**).
6. Пауза **1000 ms** (`PRIZE_FLY_DELAY_MS`).
7. Полёт приза к табу (**1200 ms**).
8. Смена панели прогресса (**1200 ms** slide).
9. Reveal новой панели (**700 ms**).

**Смена панели:**

- Текущая: `.win-progress-panel--exit`, `translateX(calc(-100% - 20px))`, **1200 ms**.
- Новая: `.win-progress-panel--enter` → `.is-active`, slide с правой стороны.

## 7. Розыгрыш (вкладка Raffle)

### 7.1. Карусель — `updateCardTransforms()`

Интерполяция по `progress` (0 = центр, 1 = боковая карточка):

| Параметр | Центр (0) | Бок (1) |
|----------|-----------|---------|
| `scale` | 1 | 0.9 |
| `translateY` | 0 px | 48 px |
| `opacity` | 1 | 0.75 |
| `rotate` | centerRotate × (1−p) + offset/step × 4° | — |

**centerRotate по карточкам:** `[+1°, −2°, 0°]`.

Обновление: на `scroll` (RAF) + `scrollend` (debounce **120 ms**).

### 7.2. Переворот карточки тапом — `toggleRaffleCardFlip()`

- **750 ms**, `cubic-bezier(0.4, 0, 0.15, 1)`.
- Накопительный `rotateY` (+180° за тап).
- Класс `.is-flipped` при нечётном числе полуоборотов.
- Visual на лицевой стороне при flip: `scale(0.9)` от `right bottom`, transition только при `[data-flip-animating="true"]`.

### 7.3. Участие в розыгрыше — `runRaffleParticipate()`

| # | Стадия | Длительность | Easing / параметры |
|---|--------|--------------|-------------------|
| 1 | Задержка | 200 ms | — |
| 2 | Scale + выравнивание | 600 ms | wrap: `cubic-bezier(0, 0, 0.4, 0.9)`, scale **1.05**, rotate **0°** |
| 3 | Flip 0° → 180° | 500 ms | `cubic-bezier(0.4, 0, 1, 1)` |
| 4 | Смена контента (участвую) | — | — |
| 5 | Flip 180° → 360° | 1200 ms | `cubic-bezier(0.4, 0.1, 0.2, 1)` |
| 6 | Нормализация rotation | — | `normalizeRaffleFlipRotation()` |
| 7 | Пауза | 400 ms | — |
| 8 | Возврат в позицию карусели | 300 ms | `cubic-bezier(0.45, 1.45, 0.48, 1)`, rotate → centerRotate |
| 9 | Искры | на 150 ms return | `spawnPrizeParticles`, sparkOnly |

**Итого активной анимации:** ~3200 ms (без задержки 200 ms ≈ 3000 ms).

На обороте во время flip показывается `.raffle-event-card__turnover`.

### 7.4. Intro при первом входе — `runRaffleIntroAnimation()`

**Предусловие:** задержка `RAFFLE_INTRO_ENTRY_DELAY_MS` = **400 ms**.

**Начальное состояние:**

- Позиция: центр экрана, `scale: 0.8`, `opacity: 0`.
- Карточка 0: flip **180°**, turnover-entry на обороте.
- Карточки 1–2: flip **0°**.

#### Стадия 1 — 900 ms

- Easing: `cubic-bezier(0, 0, 0.4, 0.9)`.
- Все слоты едут в центр карусели.
- Карточка 0: `opacity → 1`, flip → 0°, visual scale → 1.
- Карточки 1–2: остаются `opacity: 0`.

#### Пауза — 400 ms

#### Стадия 2 — 1200 ms

- Easing: `cubic-bezier(0.35, 1.3, 0.25, 1)`.
- Раскладка в позиции карусели (параметры как у `getRaffleCarouselWrapTransform`).
- Боковые карточки: `opacity 0 → 1`.
- Пагинация: `opacity 0 → 1`.

**Итого с задержкой:** 400 + 900 + 400 + 1200 = **2900 ms**.

### 7.5. Shimmer на кнопке «Участвовать»

- Класс `.is-shimmer::after`.
- **380 ms**, `ease-out`.
- Градиентная полоса: `translate(-55%, 55%)` → `translate(55%, -55%)`.
